# Capture baseline evidence

`pnpm capture:baseline` converts the latest completed NexusReach and SignalDraft pilot runs into a private, machine-readable quality baseline. It does not start either product, replay a flow, contact a provider, or copy media into Git. It inspects the already-retained private pilot artifacts and writes `tmp/capture-baseline/baseline-report.json` with mode `0600`.

## Safety boundary

- The registered pilot and workflow identities come from `capture-pilots/baseline.json`.
- Latest pointers contribute only a runtime-validated run ID. Their stored absolute paths are ignored; report and checkpoint paths are derived below the registered pilot root.
- Every inspected media or caption artifact must be a regular non-symlink file below that run's `private-artifacts` directory.
- FFprobe is invoked with an argument array, timeout, and bounded output.
- The emitted report contains artifact IDs, hashes, byte counts, media probes, interaction counts, coverage counts, and repository-evidence hashes. It omits local paths, object keys, local URLs, verification payloads, caption text, credentials, and product content.
- A failed threshold produces a stable finding code and a nonzero CLI exit. The command does not alter or delete pilot artifacts.

## Version 1 acceptance thresholds

The committed configuration currently requires:

- 5–180 seconds per clip;
- at least 1280×720 for normalized landscape clips;
- exactly 1080×1920 for vertical renders;
- H.264 video and AAC audio on vertical renders;
- 24–60 frames per second;
- a visible pointer with at least 250 ms movement presentation;
- character typing delay from 15–80 ms;
- an editable WebVTT track containing at least one cue;
- all sampled vertical frames to pass the existing informative-frame QA;
- completed, verified attempts for every registered workflow; and
- complete declared-goal and approved-flow coverage with no uncovered or blocked entries.

These thresholds establish a regression baseline. They do not prove human comprehension, text legibility after mobile composition, caption collision avoidance, or exhaustive product coverage; later quality and coverage phases must add those measurements.

- seven of seven registered workflows verified;
- seven normalized landscape clips;
- seven 1080×1920 vertical renders;
- seven editable caption tracks;
- zero threshold failures; and
- zero warnings.
